HTML 문서는 태그와 정보로 구성되어 있습니다. 태그는 정보의 유형이나 속성 등을 나타내며,
<TAG>정보</TAG> 와 같은 형태로 표현됩니다.
슬래쉬가 없는 앞쪽은 여는 태그라고 부르고, 슬래쉬가 있는 뒤쪽은 닫는 태그라고 부릅니다.
정보는 단순한 텍스트가 될 수도 있으며, 또 다른 HTML tag일 수도 있습니다.
정보가 포함되지 않은 특정 태그들은 닫는 태그를 사용하지 않아도 됩니다. 네. <br>태그처럼 말이죠.
이제 HTML의 기본적인 구조를 정의하는 태그들에 대해 알아봅시다.
  • <html>태그는 사용자에게 최종적으로 하나의 웹페이지를 제공하는 데 필요한 모든 정보를 포함하고 있는 최상위 태그입니다.
  • <head>태그는 웹페이지의 메타데이터를 기술하는 태그입니다. 메타데이터란, 데이터에 대한 데이터입니다.
    <title>, <head>, <link> 등의 태그들이 <head> 태그 안에서 사용됩니다.
  • <title> 태그는 <head>태그의 하위에 위치하며 웹페이지의 제목을 나타냅니다. 주로 브라우저의 탭에서 확인할 수 있습니다.
  • <body>태그는 웹페이지의 본문으로 실제로 웹페이지에 포함되어야 할 컨텐츠가 기술되며, 화면을 그리는 태그라고 할 수 있습니다.
    앞서 배운 <li>, <p>, <a>, <img> 등의 태그들은 모두 <body> 태그 안에서 사용됩니다.
  • <style>태그는 웹페이지의 스타일을 지정하는 태그로 웹페이지를 보다 사용자 친화적이며 아름답게 꾸밀 수 있게 합니다.
  • <script>태그는 웹페이지의 동작과 기능을 구현하는 태그로 사용자에게 더 다양한 상호작용을 제공합니다.

HTML의 게층구조

HTML 태그들은 서로 계층적인 상관관계를 갖습니다.
앞서 HTML tag의 정보는 또 다른 HTML tag 가 될 수 있다고 말씀드렸습니다.
HTML tag 안에 다시 HTML tag가 포함되어 있다면 그 두 HTML tag는 부모자식 관계입니다.
떡잎유치원은 짱구와 훈이의 부모입니다. 짱구와 훈이는 떡잎유치원의 자식입니다. 그리고 짱구와 훈이는 부모자식관계가 아닌 친구입니다. 그런데 유치원과 원생의 관계는 부모자식이라기 보다는 포함관계가 아닌가요?
