HTML 작성하기

HTML문서는 태그와 정보로 구성되어 있습니다. 태그는 정보의 유형이나 속성 등을 나타내며,
<TAG>정보</TAG> 와 같은 형태로 표현됩니다.
슬래쉬가 없는 앞쪽은 여는태그라고 부르고, 슬래쉬가 있는 뒷쪽은 닫는태그라고 부릅니다.
정보는 단순한 텍스트가 될 수도 있으며, 또 다른 HTML tag일 수도 있습니다.
정보가 포함되지 않는 특정 태그들은 닫는태그를 사용하지 않아도 됩니다. 네. <br> 태그처럼 말이죠.
이제 HTML의 기본적인 구조를 정의하는 태그들에 대해 알아봅시다.
  • <html>태그는 사용자에게 최종적으로 하나의 웹페이지를 제공하는데 필요한 모든 정보를 포함하고 있는 최상위 태그입니다.
  • <head>태그는 웹페이지의 메타데이터를 기술하는 태그입니다. 메타데이터란, 정확하게 번역하기는 어렵지만 보조적인 정보라고 생각하시면 됩니다.
    <title>, <meta>, <link> 등의 태그들이 <head> 태그 안에서 사용됩니다.
  • <title>태그는 <head>태그의 하위에 위치하며 웹페이지의 제목을 나타냅니다. 주로 브라우저의 탭에서 확인 할 수 있습니다.
  • <body>태그는 웹페이지의 본문으로 실제로 웹페이지에 포함되어야 할 컨텐츠가 기술되며, 화면을 그리는 태그라고 할 수 있습니다.
    앞서 배운 <li>, <p>, <a>, <img> 등의 태그들은 모두 <body> 태그 안에서 사용됩니다.
  • <style>태그는 웹페이지의 스타일을 지정하는 태그로 웹페이지를 보다 사용자 친화적이며 아름답게 꾸밀 수 있게 합니다.
  • <script>태그는 웹페이지의 동작과 기능을 구현하는 태그로 사용자에게 더 다양한 상호작용을 제공합니다.

HTML의 계층구조

HTML 태그들은 서로 계층적인 상관관계를 갖습니다.
앞서 HTML tag의 정보는 또 다른 HTML tag가 될 수 있다고 말씀드렸습니다.
HTML tag 안에 다시 HTML tag가 포함되어 있다면 그 두 HTML tag는 부모자식 관계입니다.
떡잎유치원은 짱구와 훈이의 부모입니다. 반대로 짱구와 훈이는 떡잎유치원의 자식입니다. 마지막으로 짱구와 훈이는 부모자식 관꼐가 아닌 친구입니다.

HTML 요소의 속성

모든 HTML 요소들은 속성을 갖고 있습니다. 어느 태그에나 존재하는 공통적인 속성들도 있고, 특정 태그에만 존재하는 속성들도 있습니다.
많이 보신 id와 class도 물론 속성입니다.
